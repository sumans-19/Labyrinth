# Contributing to Labyrinth Forge

First off, thank you for considering contributing to Labyrinth Forge! It's people like you that make the security community so great.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug and include steps to reproduce.

### Suggesting Enhancements
- Open an issue with a clear title and description.
- Explain why this enhancement would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue that pull request!

## Style Guide
- Follow PEP 8 for Python code.
- Use functional components and hooks for React.
- Ensure all new features are documented in the README.

## Security Policy
If you find a security vulnerability, please do NOT open a public issue. Instead, email security@labyrinthforge.com
